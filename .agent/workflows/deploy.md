# Deploy Workflow (Thai Akha Cherry 2026)

Questo workflow è l'hub centrale "Super Power" per gestire in modo sicuro e rapido il ciclo di sviluppo, il rilascio in produzione, e la configurazione dell'infrastruttura (Firebase, Supabase, Cloudflare) di "Thai Akha Kitchen 2026". Tutte le documentazioni tecniche, comandi di setup e know-how sono state raggruppate qui.

## 🏗 Struttura Progetto & Requisiti
- **Monorepo pnpm**: `packages/front` (visitor app), `packages/admin` (staff app), `packages/shared` (librerie comuni e Cherry AI Unified Brain).
- **Core Stack**: React/Vite, Node >= 18, pnpm >= 9.
- **Integrazioni**: Firebase Hosting, Supabase DB & Edge Functions, Google Gemini, Cloudflare Workers (per Open Graph SEO).
- **Env Vars Richieste** (in `packages/front/.env.local` e `packages/admin/.env.local`):
  - `VITE_GEMINI_API_KEY`, `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`, `VITE_GOOGLE_MAPS_API_KEY`.

## 🚀 Azioni di Rilascio Principali

### 1. Run Locally (`/deploy run locally`)
Avvia l'ambiente di sviluppo locale.
// turbo-all
1. Ferma eventuali server attivi: `lsof -ti :3000,3001 | xargs kill -9`
2. Installa dipendenze (se package.json modificato): `pnpm install`
3. Avvia i server in parallelo: `pnpm run dev` (Front e Admin simultaneamente)

### 2. Push GitHub (`/deploy push github`)
Esegue il commit e il push delle ultime modifiche sul repository centrale.
// turbo-all
1. `git add .`
2. `git commit -m "Sync & Update [Autogenerated via Agent]"`
3. `git push origin main`

### 3. Deploy Firebase (`/deploy deploy firebase`)
Compila l'applicazione monorepo e pubblica su Firebase Hosting i pacchetti `frontend` e `admin`. L'alias di default è `thai-akha-app-2026`.
// turbo-all
1. Compila i pacchetti (verifica TS stricta): `pnpm run build`
2. Esegue il deploy: `pnpm dlx firebase-tools deploy --only hosting --project default`
*(I target sono mappati nel `firebase.json`: `frontend` -> `dist`, `admin` -> `dist`)*

## ⚙️ Gestione Infrastruttura Backend (Supabase & Cloudflare)

Comandi per il deploy delle funzioni Open Graph Meta-tags per condividere link sui bot visivi (Facebook, Whatsapp).

### 4. Deploy Supabase Edge Functions (`/deploy supabase edge`)
// turbo-all
1. Esegui la migrazione DB SEO (se non fatta): `supabase db push`
2. Deploy the function per rendering HTML bot: `supabase functions deploy og-meta-tags --no-verify-jwt`
*(Project ref su MCP locale: `mtqullobcsypkqgdkaob`)*

### 5. Deploy Cloudflare Worker (`/deploy cloudflare worker`)
// turbo-all
1. Deploy via Wrangler del file di root `cloudflare-worker.js` (assicurati che l'URL punti alla funzione supabase corretta): `pnpm dlx wrangler deploy --path cloudflare-worker.js` 
*(Route Cloudflare: `thaiakhakitchen.com/*`)*

## 🔒 Troubleshooting Frequente (QA)

### Errori Risoluzione Build
- **Errore Build Typescript**: `pnpm run build` fallisce bloccando il deploy se ci sono refactors incompleti in shared. Passare per la detection a strati:
  `cd packages/shared && npx tsc --noEmit` per isolare il core.

### Errori Cherry AI App
- **Voce Interrotta/Errore Chrome**: Il modello Live API Google vuole audio processor; funziona ESCLUSIVAMENTE via HTTPS o su `localhost`. Controlla sempre che `audio-processor.js` sia copiato e raggiungibile dalla `public/`.
- **Supabase Session 400**: Se la tabella `chat_sessions` invia un HTTP 400 Bad Request, assicuratevi che `session_token` sia *nullable*, e le colonne `metadata` (jsonb) e `type` (text) siano presenti. 

### Errori Infrastruttura SEO
- Se i metadata non escono bene sul Facebook Debugger: testare l'Edge function su cURL per vedere le risposte flat HTML: `curl "https://mtqullobcsypkqgdkaob.functions.supabase.co/og-meta-tags?path=/history/test"`.
